用 Hugo 和 Github Page 搭建自己的博客网站

利用Hugo框架和白嫖的github.io网站搭博客

Oct 14, 2024
阅读时长: 4 分钟

利用Hugo作为框架，采用的Theme为Stack，申请Github Page的服务器和域名，部署自己的静态博客网站。

原理

服务器和域名用github page的，Hugo是一个用go编写的的静态网页生成器，我的理解是用户负责提供markdown等格式的源文件，Hugo负责生成有一定结构的文件，包含html等，部署到github.io上后即可获得精美的个人博客。

环境

  • windows11 WSL2
  • Linux发行版本是 unbuntu 22.04
  • go 1.23.2
  • hugo v0.135.0

安装

我用的stack theme似乎有可以不用本地安装的方法，具体是用了github的codespace详情见stack作者的Readme，不过我采用的是本地搭建的方式，需要安装Git，GO和Hugo。安装网站分别是

  • https://git-scm.com/book/zh/v2/%E8%B5%B7%E6%AD%A5-%E5%AE%89%E8%A3%85-Git
  • https://go.dev/doc/install
  • https://gohugo.io/installation/

值得一提的是go的安装需要先下载然后用

rm -rf /usr/local/go && tar -C /usr/local -xzf go1.23.2.linux-amd64.tar.gz

安装，如果出现权限问题的报错，试一试sudo，记得在两个语句前都加上

sudo rm -rf /usr/local/go && sudo tar -C /usr/local -xzf go1.23.2.linux-amd64.tar.gz

以及hugo的安装记得安装extension版本。

操作

主题模板下载

正常来说可以按照hugo官方文档的quick start一步一步来，教程很详细，但stack主题和正常不太一样，我的做法是自己结合stack的官网教程摸索出来的，并没有完全按照hugo官方教程来。

首先

git clone https://github.com/CaiJimmy/hugo-theme-stack-starter.git

将stack-starter的代码复制到本地，然后直接

1
2

cd hugo-theme-stack-starter/  
hugo

两行命令，网站就构建好了，打开浏览器网址栏输入localhost:1313，就可以发现已经有一个像模像样的雏形网站了。这个时候进一步观察发现目录下多了两个文件夹public/和resources/这里面就是hugo帮我们生成的一堆html文件以及相关素材，网站展示也只需要pubulic/文件夹，其余的可以留在本地。

修改网站和标题

在config/_default/config.toml里将网址和title改成自己的

客制化

默认的头像肯定是要换的，但由于stack-starter是用到了hugo的module，修改不太一样，经过我的一番探索，可以把 ~/.cache/hugo_cache/modules/filecache/modules/pkg/mod/github.com/!cai!jimmy/hugo-theme-stack/v3@v3.27.0/目录下的 asserts/ archetypes/ layout/文件夹copy到工作目录，i.e.hugo-theme-stack-starter/ 目录下，然后加上读和写的权限，以layout/为例

1
2
3

cp -r layouts/ ~/mysite/hugo-theme-stack-starter/
cd ~/mysite/hugo-theme-stack-starter/
sudo chmod -R a+rw ./layouts/

这样就可以在asserts/img/里放入头像照片，在archetypes里修改post，categories还有tag的默认模板了。多数客制化的内容可以在config/_default/param.toml里修改，具体参数可以参见stack config页面，内容很全。

添加文章和类别

  • 添加文章：hugo new content content/post/<your-title>/index.md
  • 添加类别: hugo new content content/categories/<category-name>/_index.md

申请github.io并设置发布源

关于申请github page ，github document讲的很详细，不多赘述。关于配置发布源，文档也有详细描述，我尝试了一下github action没有成功，加上用写好的脚本部署本身也不复杂，就没有选择通过action发布，而是分支发布，分支发布的具体操作见上述文档。

当编辑完markdown并保存后，可以运行hugo官方提供的一键部署代码。 (ps:我在hugo官网上没找到这个，粘贴自另一位大佬的博客)

#!/bin/sh

# 任一步骤执行失败都会终止整个部署过程
set -e

printf "\033[0;32mDeploying updates to GitHub...\033[0m\n"

# 构建静态内容
hugo # if using a theme, replace with `hugo -t <YOURTHEME>`

# 切换到 Public 文件夹
cd public

# 添加更改到 git
git add .

# 提交更改
msg="rebuilding site $(date)"
if [ -n "$*" ]; then
msg="$*"
fi
git commit -m "$msg"

# 推送到远程仓库
git push origin main

添加执行权限后，chmod +x deploy.sh。就可以运行了./deploy.sh

脚本会自动将public文件夹推送到github page。过一会儿就可以看到自己的主页了。

To use this feature, add `links` section to frontmatter.

This page's frontmatter:

```yaml
links:
  - title: GitHub
    description: GitHub is the world's largest software development platform.
    website: https://github.com
    image: https://github.githubassets.com/images/modules/logos_page/GitHub-Mark.png
  - title: TypeScript
    description: TypeScript is a typed superset of JavaScript that compiles to plain JavaScript.
    website: https://www.typescriptlang.org
    image: ts-logo-128.jpg
```

`image` field accepts both local and external images.
