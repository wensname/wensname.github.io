终端美化

Oct 17, 2024
阅读时长: 4 分钟

简单介绍一下美化终端~

环境

  • 操作系统：windows11 / WSL2（ubuntu22.04）
  • 终端：windows terminal
  • shell: windows powershell / fish

字体

安装

一个好看的终端免不了需要一些图案，而默认的字体往往不含有这些图案，这时候就需要用到Nerd Fonts，Nerd Fonts本身不是一个新的字体，而是将图案打补丁到原有字体上。可以去官网找自己喜欢的字体Nerd Fonts - Iconic font aggregator, glyphs/icons collection, & fonts patcher。我找的是Hack Nerd Font。安装路径是nerd-fonts/patched-fonts/Hack/Regular/HackNerdFont-Regular.ttf at master · ryanoasis/nerd-fonts (github.com)，直接下载ttf文件打开后点击安装即可。

验证字体安装

安装过后，命令行里就支持显示这些小图标了，为了测试是否成功，可以到这个网址：www.nerdfonts.com/cheat-sheet

点击 Show All Icons 按钮，选择一个图标，点击右上角的 Copy Icon，然后粘贴到 Terminal 命令行里。出现图标而不是乱码即为成功

设置终端背景

win+R打开windows运行框，输入wt，弹出windows terminal窗口，Ctrl+,打开设置页面，在 “默认值/外观/字体” 中选择字体，还可以在“外观”页面中自行设置背景和透明度等。

Windows 11 的 Powershell

安装并配置oh-my-posh

安装

在命令行一句

winget install JanDeDobbeleer.OhMyPosh -s winget

即可搞定。

配置

首先，需要搞清楚用的什么终端，命令行中输入 oh-my-posh get shell 即可获得终端类型，再根据oh-my-posh官网选择刚刚看到的shell进行对应的操作，这里以 powershell 为例。

notepad $PROFILE打开配置文件，如果没有系统会自动新建一个，然后将下面这行加到文件最后一行

1

oh-my-posh init pwsh | Invoke-Expression

然后运行. $PROFILE 加载一下该文件

配置好之后，你会发现关掉终端重新进去就直接是默认主题的样子，这个时候如果想要重新挑选一个主题，可以输入Get-PoshThemes 来看所有预设的主题，可以挑选一个自己喜欢的，我个人比较喜欢clean-detailed的风格，用notepad $PROFILE打开配置文件，将刚刚加的那一行改为如下：

1

oh-my-posh init pwsh --config C:\Users\<your name>\AppData\Local\Programs\oh-my-posh\themes\clean-detailed.omp.json | Invoke-Expression

将改为自己的名字即可，后面的json文件挑自己喜欢的用。

然后运行. $PROFILE就会发现主题已经变了，至此我们windows的终端美化就做好了。

WSL2中的fish

安装brew。。

官方教程让用brew安装oh-my-posh，那咱们就装吧。我用的fish，用下面这个命令不知道为什么总装不上

1

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

当然，改成fish的

1

/bin/bash -c (curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)

也试过，没什么用。无奈之下，终端输入/bin/bash退出fish进入bash，这时候运行第一行命令就ok了。

安装完毕后记得添加环境变量。将

1

eval (/home/linuxbrew/.linuxbrew/bin/brew shellenv)

加入到~/.config/fish/config.fish 文件的最后一行。完事之后source一下就ok了。

安装并配置oh-my-posh

安装

brew install jandedobbeleer/oh-my-posh/oh-my-posh

配置

将 oh-my-posh init fish | source 加入到 ~/.config/fish/config.fish 文件的最后一行即可

客制化

将上面一行改为

oh-my-posh init fish --config /mnt/c/Users/<your name>/AppData/Local/Programs/oh-my-posh/themes/clean-detailed.omp.json | source 即可

注意到wsl实际上是可以访问本地windows系统的文件的，文件路径就为上图所示。

至此终端美化就完成了。

番外

踩了一个坑

之前安装fish的时候直接想当然sudo apt install fish,也没报什么错，但实际上apt默认用的版本是3.3.1+ds-3，这个版本的fish command-line命令没有–is-valid选项，而oh-my-posh的某个代码用到了这个选项，导致每次键入命令都会出现几行报错很难受。解决方法是按照官方install上执行如下命令安装最新版本

1
2
3

sudo apt-add-repository ppa:fish-shell/release-3
sudo apt update
sudo apt install fish

把版本升级了就好了。最终结果展示：
